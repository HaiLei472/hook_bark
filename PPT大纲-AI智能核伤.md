# AI 智能核伤——PPT 大纲（11 页）

## 第一部分：业务痛点

### Slide 1：封面
- 标题：AI 智能核伤——车险人伤鉴定智能审核解决方案
- 副标题：基于大模型的伤残鉴定报告智能核验
- 提案人 / 日期

---

### Slide 2：骗保链条——三方串通分账
- 车险人伤理赔链路：出险 → 就医 → **伤残鉴定** → 核赔审核 → 赔付
- 伤残等级每差一级，赔偿金额差 10 万+
- 骗保链条流程图：
  ```
  黄牛拉拢伤者 ── "帮你多拿赔偿，事成后分成"
        │
        ▼
  引荐"合作"的鉴定机构 ── 出具虚高伤残等级报告
        │
        ▼
  保险公司超赔 → 三方分账
  ```
- 痛点本质：理赔员面对海量案件，难以逐案甄别鉴定报告合理性

---

### Slide 3：理赔员的四大困境
- 四宫格：
  - **审不过来**：日均 20-30 件，每件审核 20 分钟+
  - **审不深**：缺医学/法医临床学背景，难判条款适用性
  - **审不准**：关键数值散落，人工比对易遗漏
  - **无参照**：同类伤情应评几级？缺数据支撑

---

## 第二部分：解决思路

### Slide 4：方案定位——AI 智能核伤
- 一句话：在理赔系统中嵌入 AI 能力，一键核验鉴定报告
- 三个原则：
  - 不改流程——理赔员工作流不变
  - 不替决策——AI 只标疑，不判定
  - 不增负担——一键触发，秒级返回
- 核心价值：降低虚高鉴定导致的超额赔付，直接减损

---

### Slide 5：核心架构——三阶段流水线
- 流程图：
  ```
  ┌──────────┐    ┌──────────┐    ┌──────────┐
  │  阶段一   │    │  阶段二   │    │  阶段三   │
  │ 材料结构化 │───▶│ 多维校验  │───▶│ 综合结论  │
  │  抽取     │    │          │    │          │
  └──────────┘    └──────────┘    └──────────┘
      LLM           规则+LLM          LLM
  ```
- 阶段一：LLM 读懂鉴定报告 + 病历，抽取结构化字段
- 阶段二：三维度交叉校验（条款匹配度 / 关键数值 / 文书一致性）
- 阶段三：生成理赔员可读的审核结论 + 风险评分

---

### Slide 6：核心能力——条款匹配度校验
- 真实场景演示：
  - 鉴定报告：左肩关节功能丧失 50%，引用条款 5.9.6.1，评定 9 级
  - 病历诊断：左肩部软组织挫伤
  - AI 判断：软组织挫伤无关节结构损伤基础 → 🔴 不匹配
- 这是规则引擎做不到的——需要 LLM 语义理解
- 数值校验举例：鉴定测 45° vs 出院记录 80°，差 35° 标红

---

### Slide 7：审核结论示例
- 输出样例卡片：
  - 风险评分：75/100（高风险）
  - 🔴 伤残等级存疑：鉴定 9 级，但病历为软组织挫伤
  - 🔴 关节活动度不一致：鉴定 45° vs 出院记录 80°
  - 建议操作：启动调查流程
- 评分规则：红色 +20/项，黄色 +5/项
- 技术底座：产险大脑（DeepSeek-v4）私有化部署，数据不出内网

## 第三部分：落地路径

### Slide 8：智能核伤 Demo——一个真实可疑案件

**案件背景**：伤者李某，女，32 岁，2025 年 10 月交通事故致左肩受伤，鉴定机构评定为 9 级伤残。

**输入材料**（理赔员上传）：
```
【鉴定报告】
伤者：李某，女，32岁
受伤原因：2025年10月1日交通事故致左肩受伤
鉴定日期：2025年12月15日
伤残等级：九级
依据条款：《人体损伤致残程度分级》5.9.6.1
        "一上肢三大关节功能丧失50%以上"
检查所见：左肩关节前屈50°，外展45°
        左肩关节功能丧失约60%

【病历材料】
入院诊断：左肩部软组织挫伤
出院诊断：左肩部软组织挫伤
手术记录：无
影像报告：X线片未见骨折，超声未见肩袖撕裂
出院情况：左肩活动自如，无功能障碍
住院天数：3天
```

**AI 智能核伤输出**：

```
┌───────────────────────────────────────────┐
│  AI 核伤结论                                │
│  案件编号：CASE-2025-1031                   │
│  风险评分：85/100（高风险）                  │
├───────────────────────────────────────────┤
│  🔴 条款匹配度存疑                           │
│  鉴定引用条款 5.9.6.1（关节功能丧失50%以上） │
│  但病历诊断为"软组织挫伤"，无关节结构损伤    │
│  证据，缺乏条款所要求的器质性损伤基础        │
│  原文摘引："左肩部软组织挫伤"                │
├───────────────────────────────────────────┤
│  🔴 关节活动度数值不一致                     │
│  鉴定报告：左肩前屈50°                       │
│  出院记录：左肩活动自如                      │
│  逻辑矛盾：3 天住院 + 软组织挫伤 + 出院活动  │
│  自如，与鉴定测量的严重活动受限不符          │
├───────────────────────────────────────────┤
│  🔴 住院天数异常                             │
│  住院 3 天即出院，符合轻伤治疗模式           │
│  与 9 级伤残应有的严重伤情不符               │
├───────────────────────────────────────────┤
│  建议操作：启动调查流程                      │
│  推荐调查方向：                              │
│  1. 复核鉴定机构资质及历史评级偏差           │
│  2. 调取事故现场及就医当日影像资料           │
│  3. 联系伤者复查肩关节功能                   │
└───────────────────────────────────────────┘
```

**理赔员反馈**：原需 20 分钟通读材料 + 自己判断，现 3 分钟即可定向确认 3 个红色疑点，决定启动调查。

### Slide 9：实施路线与 MVP 范围
- 三阶段时间轴：
  - **MVP（5 天）**：3-5 类高频伤情 + 核心条款 + 文本输入
  - **V1.0**：全覆盖条款 + PDF 直传 + 影像辅助
  - **V2.0**：历史案件检索 + 鉴定机构风险画像
- MVP 交付：理赔系统内嵌「AI 核伤」按钮 + 50-100 例回溯测试

---

### Slide 10：效果衡量
- 四个指标卡片：
  - 高风险识别率 ≥ 85%
  - 误伤率 ≤ 15%
  - 单案处理效率提升 ≥ 50%
  - 减损金额（直接 ROI）

## 第四部分：风险与边界

### Slide 11：风险应对与能力边界
- 风险应对表：
  | 风险 | 应对 |
  |------|------|
  | LLM 抽取准确率 | Prompt 强调原文摘录 + 后置正则校验 |
  | 理赔员信任度 | 每条风险标注原文出处；AI 只标疑不判定 |
  | 鉴定机构对抗升级 | 校验逻辑保密 + 定期更新 prompt |
  | 合规风险 | 定位辅助工具，决策可追溯还原 |
- 三条能力边界：
  1. 不做最终赔付判定——决策权在理赔员
  2. 不替代医学鉴定——只校验已有报告合理性
  3. 不处理手写体病历——MVP 仅支持印刷体/电子病历

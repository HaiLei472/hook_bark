# AI 智能核伤——车险人伤鉴定智能审核解决方案

## 一、背景与痛点

### 1.1 业务背景

车险人伤理赔中，伤残等级评定是决定赔付金额的核心环节。根据《人体损伤致残程度分级》，伤残分为 1-10 级，每级对应的赔偿金额差异巨大——以一名 40 岁城镇伤者为例，9 级（赔付系数为20%）和 10 级（赔付系数为10%）之间的赔偿差额可达 10 万元以上。

当前行业流程中，伤残鉴定由第三方司法鉴定机构完成。保险公司理赔员收到鉴定报告后，需结合病历、诊断证明等材料逐份审核，判断鉴定结论是否合理。

### 1.2 核心痛点：三方串通骗保

行业面临的最大风险是**"黄牛 + 伤者 + 鉴定机构"的骗保链条**：

```
交通事故发生
     │
     ▼
黄牛主动接触伤者 ── "帮你多拿赔偿，事成后分成"
     │
     ▼
引导伤者去"合作"的鉴定机构
     │
     ▼
鉴定机构出具虚高伤残等级报告（如软组织挫伤评为关节功能丧失级别）
     │
     ▼
保险公司超赔 → 三方分账
```

**痛点本质**：理赔员面对海量案件，仅凭个人经验难以逐案甄别鉴定报告的合理性。鉴定报告是专业文书，理赔员缺乏医学和法医临床学背景，对关节活动度测量值、条款适用性等专业内容力不从心。

### 1.3 当前困境

| 困境 | 表现 |
|------|------|
| **审不过来** | 一个理赔员日均处理 20-30 件人伤案件，每件伤残案件审核耗时 20 分钟以上 |
| **审不深** | 缺少医学专业知识，难以判断"半月板 III 度损伤"与"9 级伤残"之间的因果关系是否成立 |
| **审不准** | 关键数值（关节活动度、肌力等级）散落在鉴定报告和病历的不同位置，人工比对极易遗漏 |
| **无历史参照** | 同类伤情的正常评级区间是什么？理赔员缺少数据支撑 |

## 二、解决方案概述

### 2.1 产品定位

在保险公司现有理赔系统中，新增**"AI 智能核伤"功能模块**——并非另建一套独立系统，而是作为理赔工作台的一个能力增强。

**触发方式**：理赔员打开含伤残鉴定报告的案件 → 点击「AI 核伤」→ 系统自动输出审核结论。

**核心原则**：AI 只做"标疑"，不做"判定"（在AI核伤准确率提高到90%时，可让AI全权判定）。最终是否拒赔、是否调查，由理赔员决定。

### 2.2 核心价值

> **一句话：降低虚高鉴定导致的超额赔付，直接减损。**

- 对理赔员：从"20 分钟通读材料自己判断" → "5 分钟定向复核 AI 标出的 3-5 个风险点"
- 对保险公司：压缩虚高鉴定空间，增加骗保成本
- 对行业：让黄牛"合作鉴定机构"的套路失效

## 三、核心技术设计

### 3.1 整体架构

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│  阶段一   │    │  阶段二   │    │  阶段三   │
│ 材料结构化 │───▶│ 多维校验  │───▶│ 综合结论  │
│  抽取     │    │          │    │          │
└──────────┘    └──────────┘    └──────────┘
    LLM           规则+LLM          LLM
```

### 3.2 阶段一：材料结构化抽取

将鉴定报告和病历材料输入大模型（产险大脑 DeepSeek-v4），自动抽取为结构化字段。

**从鉴定报告中抽取**：
- 伤者基本信息（年龄、性别）
- 受伤原因与部位
- 评定伤残等级
- 引用的《人体损伤致残程度分级》条款编号及原文
- 关键临床指标数值（关节活动度、肌力等级、影像学发现）

**从病历材料中抽取**：
- 入院/出院诊断
- 手术记录
- 影像报告关键描述
- 出院时功能恢复状态
- 医嘱

**设计原则**：宁可多抽不漏。即使部分字段暂时用不上，也比校验时发现缺数据好。

### 3.3 阶段二：多维校验

从三个维度对抽取结果进行交叉核验：

**维度 1：条款匹配度校验**

核心逻辑——病历诊断的伤情是否真的能支撑鉴定报告所引用的伤残条款？

- 用 LLM 判断伤情与条款的因果关系
- 实例：鉴定报告引用"一上肢三大关节功能丧失 50% 以上"评为 9 级，但病历仅为"左肩软组织挫伤"——无关节结构损伤，条款引用不成立
- 这是最核心的校验维度，需要 LLM 的语义理解能力而非简单关键词匹配

**维度 2：关键数值校验**

同名客观指标在鉴定报告和病历间的数值交叉对比：

- 关节活动度：鉴定报告测量值 vs. 出院记录记载值（差值 >15° 标红）
- 肌力等级：鉴定报告 vs. 病历中神经科检查结果
- 影像学证据一致性

此维度用规则引擎实现，确定性高，不需 LLM。

**维度 3：文书一致性校验**

- 伤者基本信息是否一致
- 受伤原因描述是否与事故认定书一致
- 鉴定机构资质是否在有效期内

### 3.4 阶段三：综合结论生成

汇总三个维度的校验结果，生成理赔员可读的审核结论：

```
┌─────────────────────────────────────────┐
│ AI 核伤结论                              │
│                                         │
│ 风险评分：75/100（高风险）                │
│                                         │
│ 🔴 伤残等级存疑                           │
│ 鉴定评定 9 级（条款 5.9.6.1），但病历诊断  │
│ 为左肩软组织挫伤，无关节结构损伤证据        │
│                                         │
│ 🔴 关节活动度数值不一致                    │
│ 鉴定报告测 45°，出院记录记载约 80°，       │
│ 差值 35° 远超 15° 阈值                    │
│                                         │
│ 建议：启动调查流程                         │
└─────────────────────────────────────────┘
```

**风险评分规则**：
- 红色异常项：+20 分/项
- 黄色异常项：+5 分/项
- 0-30 低风险（通过）| 31-60 中风险（退回要求说明）| 61-100 高风险（启动调查）

## 四、技术底座

### 4.1 大模型能力

调用公司已有的**产险大脑平台 API**（已部署 DeepSeek-v4），私有化部署，数据不出内网。

单案件 LLM 调用次数：最多 4 次（抽取 ×2 + 条款匹配度 ×1 + 结论生成 ×1）。

### 4.2 为什么 LLM + 规则而非纯规则？

| | 纯规则引擎 | LLM + 规则混合 |
|---|---|---|
| 条款匹配度判断 | 无法实现——条款用语与病历用语差异大，需语义理解 | LLM 擅长 |
| 数值对比校验 | 精准、确定性高 | 不需要 LLM（还可能出错） |
| 结论生成 | 模板化生硬 | LLM 生成自然、可读性强 |

混合架构取两者之长。

## 五、实施路线

### MVP（第 1 阶段）

| 项目 | 内容 |
|------|------|
| 覆盖伤情 | 3-5 类高频伤情（骨折、关节损伤、脊柱损伤等） |
| 覆盖条款 | 对应《人体损伤致残程度分级》核心条款 |
| 输入格式 | 鉴定报告 + 病历（印刷体/电子病历文本） |
| 验证方式 | 50-100 例历史案件回溯测试 |
| 交付形式 | 理赔系统内嵌「AI 核伤」按钮 |
| 开发周期 | 约 5 天 |

### V1.0（第 2 阶段）

- 伤残条款全覆盖
- 支持 PDF 鉴定报告/扫描件直接上传（接入 OCR）
- 增加影像胶片辅助分析（X 光/CT/MRI 图像）

### V2.0（第 3 阶段）

- **历史案件相似度检索**：将当前案件病历向量化，检索历史同伤情案件，对比伤残等级分布——"同是半月板 III 度损伤，100 例历史案件中 9 级仅占 8%，10 级占 87%"，给理赔员数据参照
- **鉴定机构风险画像**：统计每家鉴定机构的评级偏高率、某类条款的使用频率异常——从单案件审核升级为系统性风险监控

## 六、预期效果

| 指标 | 定义 | MVP 目标 |
|------|------|---------|
| **高风险识别率** | 人工最终拒赔/退回的案件中，AI 评为高风险的比例 | ≥ 85% |
| **误伤率** | AI 评为高风险但人工复核通过的比例 | ≤ 15% |
| **核赔效率** | 伤残案件单案处理时间 | 缩短 ≥ 50% |
| **减损金额** | 因识别虚高鉴定而避免的超额赔付 | 需历史数据测算 |

## 七、关键风险与应对

| 风险 | 应对 |
|------|------|
| **LLM 抽取准确率**——关键数值（关节活动度度数）抽取错误会导致后续校验失效 | Prompt 强调"原文摘录，不改写数值"；增加后置正则校验兜底 |
| **理赔员对 AI 结论的信任度** | 每一条风险标注必须附带原文摘引出处；AI 只标疑不判定，决策权留在人工 |
| **鉴定机构对抗升级**——如果鉴定机构知道 AI 的校验逻辑，可能针对性"优化"报告 | 定期更新 prompt 和校验规则；条款匹配度判断逻辑不对外公开 |
| **合规风险**——AI 参与理赔审核是否需要监管报备 | 定位为"辅助工具"，不替代人工决策；所有审核结论可追溯还原到具体材料和逻辑 |

---

## 八、总结

"AI 智能核伤"功能以**不改变理赔流程、不替代人工决策、不增加系统负担**的方式，在核赔审核环节嵌入 AI 能力，解决三方鉴定机构骗保这一行业痛点。

核心逻辑：**让 AI 读懂鉴定报告，让理赔员只关注疑点。**

技术层面依赖公司已有的产险大脑平台，开发成本可控；业务层面直接对应减损目标，ROI 清晰可衡量。
